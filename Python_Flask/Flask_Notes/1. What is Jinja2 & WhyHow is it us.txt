1. What is Jinja2 & Why/How is it used?

*What is Jinja2?**

Jinja2 is Flask's default **Template Engine**. It is a tool that turns static HTML files into **dynamic** ones. This means you can write Python-like logic (variables, loops, and conditions) directly inside your HTML.

### **Why do we use it?**

If you tried to make a separate page for every single user without Jinja2, you would have to create an individual HTML file for each person (like `ali.html`, `zainab.html`), which is impossible. With Jinja2, you **create only 1 HTML file**, and whatever data comes from the backend (Python) dynamically fits right into it.

### **How is it used? (The 3 Main Syntaxes)**

1. **`{{ ... }}` (Expressions/Variables):** Used to print (display) the value of a variable onto the HTML page.
* *Example:* `<h1>Hello, {{ user_name }}</h1>`


2. **`{% ... %}` (Statements/Logic):** Used to run loops (`for`) and conditions (`if-else`).
* *Example:* `{% if age >= 18 %} Vote {% endif %}`


3. **`{# ... #}` (Comments):** Used to write comments inside Jinja2 that will not be visible in the browser.

---

## 2. What is `render_template`?

`render_template` is a built-in function in Flask. Its job is to build a bridge between your Python code and your HTML file.

* When you write `render_template('welcome.html', user_name=name)`, Flask first looks inside your **`templates` folder** to find `welcome.html`.
* Then, it passes the Python data (`user_name=name`) to that HTML file.
* The Jinja2 engine fits that data into the HTML, and Flask sends the fully prepared web page back to the browser.

---

## 3. Line-by-Line Explanation of Your Files

Let's understand exactly what is happening in the background for both of your files, line by line.

### **File 1: `app.py` (The Python Backend)**

```python
from flask import Flask, render_template

```

* **Line 1:** We are importing the Flask framework along with the `render_template` function so that we can display our HTML files.

```python
app = Flask(__name__)

```

* **Line 2:** We are creating an object (instance) of our Flask application and naming it `app`.

```python
@app.route('/welcome/<name>')

```

* **Line 3:** This is a **Dynamic Route**. Whatever the user types in the URL after `/welcome/` (such as Ali, Zainab), Flask will capture and save it inside a variable named `<name>`.

```python
def welcome(name):

```

* **Line 4:** This is a Python function that executes whenever a user visits the URL defined above. The `name` captured from the URL is passed right into this function as a parameter.

```python
    return render_template('welcome.html', user_name=name)

```

* **Line 5:** This is the most important line! We are instructing Flask to pull `welcome.html` from the `templates` folder. Then, we create a new Jinja variable called `user_name` (used as `{{ user_name }}` in the HTML file) and assign it the value of our Python `name` variable before passing it to the HTML.

```python
if __name__ == '__main__':
    app.run(debug=True)

```

* **Line 6-7:** This is the standard way to run the script. `debug=True` means that whenever you make changes to your code and save it, the server will restart automatically, and any development errors will be shown directly on the browser.

### **File 2: `welcome.html` (The Frontend Template)**

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Welcome Page</title>
</head>

```

* **Line 1-6:** This is the standard HTML5 boilerplate structure that tells the browser this is a web page and sets its title to "Welcome Page".

```html
<body>
```

* **Line 7:** This marks the start of the main body area of the web page, which contains the content visible on the screen.

```html
    <h1>Salam, {{ user_name }}!</h1>

```

* **Line 8:** This is the **Jinja2 Variable Syntax** `{{ user_name }}`. Jinja2 will replace this placeholder with the actual value of `user_name` sent by `app.py` (for example: "Ali"). The browser ultimately receives plain HTML like `<h1>Salam, Ali!</h1>`.

```html
    <p>Welcome to our website. We are glad you are here.</p>
</body>
</html>

```

* **Line 9-11:** This displays a simple paragraph text, followed by the closing tags for the body and the HTML document.

## 4. Step-by-Step Breakdown of the Python File (`app.py`)

Let's break down exactly what each step in the `app.py` file does on the backend:

```python
from flask import Flask, render_template

```

* **Step 1:** In this line, we are importing two essential components:
* **`Flask`**: The main framework that sets up our web server.
* **`render_template`**: The specific function that instructs our Python backend how to fetch an HTML file from the `templates` folder and render it in the browser.



---

```python
app = Flask(__name__)

```

* **Step 2:** Here, we instantiate our application by creating an object named **`app`**.
* The `__name__` argument tells Flask where our main file is located, allowing it to correctly look for the `templates` and `static` folders in the same directory.



---

```python
@app.route('/welcome/<name>')

```

* **Step 3:** This defines a **Route (URL Path)**. Whenever a user visits your website, Flask checks this path.
* **`<name>` (Dynamic Part):** The angular brackets `< >` indicate that this part of the URL is dynamic. If a user visits `localhost:5000/welcome/Ali`, Flask captures the word `Ali` and stores it as a variable.



---

```python
def welcome(name):

```

* **Step 4:** This is a standard Python function. It is placed directly beneath the `@app.route` decorator because they are linked together.
* The name captured from the URL (like `Ali`) is passed straight into the `name` parameter of this function.



---

```python
    return render_template('welcome.html', user_name=name)

```

* **Step 5:** This is the output (response) that the function sends back to the browser:
* First, Flask loads the `templates/welcome.html` file.
* Next, we create a new variable called **`user_name`** (which corresponds to `{{ user_name }}` inside the HTML file) and assign it the value stored in the Python `name` variable.
* The Jinja2 engine injects this value into the HTML structure, and the final webpage displays on the screen.



---

```python
if __name__ == '__main__':
    app.run(debug=True)

```

Step 6:** This block of code initializes and starts up the web server.
debug=True`**: This flag activates debugging mode. It means you don't have to manually stop and restart your server every time you modify your code; it reloads automatically upon saving. Additionally, if your code breaks, a detailed error log will be displayed directly in the browser window to help you debug.
