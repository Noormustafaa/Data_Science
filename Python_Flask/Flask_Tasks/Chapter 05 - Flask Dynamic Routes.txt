 Chapter 05 - Flask Dynamic Routes

Introduction to Dynamic Routes

In the previous chapter, we learned about Flask Routing. Routing allows us to connect a URL with a specific Python function.

For example:

```python
@app.route("/about")
def about():
    return "About Page"
````

When a user visits:

```
http://127.0.0.1:5000/about
```

Flask finds the matching route and executes the `about()` function.

However, normal routes have a limitation. They work only with fixed URLs.

For example:

```
/about
/contact
/profile
```

These URLs are static and cannot handle changing data.

This is where **Dynamic Routes** are used.

---

# What are Dynamic Routes?

In simple words:

> Dynamic Routing allows us to create a single route that can handle different values dynamically.

Instead of creating separate routes for every user, product, or item, we create one route and pass changing values inside the URL.

Example:

```python
@app.route("/student/<name>")
def student(name):
    return "Hello " + name
```

Here:

```
<name>
```

is a dynamic part of the URL.

The value of `<name>` can change depending on the user's request.

---

# Why Do We Need Dynamic Routes?

Imagine we are creating a student management system.

We have thousands of students:

```
Ali

Ahmed

Sara

Noor

Usman
```

Without dynamic routes, we would need separate routes for every student:

```python
@app.route("/student/Ali")
def ali():
    return "Ali Profile"


@app.route("/student/Ahmed")
def ahmed():
    return "Ahmed Profile"
```

This approach is not practical.

If we have 10,000 students, we cannot create 10,000 functions.

Dynamic routes solve this problem by allowing one route to handle unlimited values.

---

# Real-Life Example of Dynamic Routing

Imagine a school office.

There is only one office:

```
School Office
```

Students do not have separate offices.

A student comes and says:

```
My name is Ali
```

The office responds:

```
Welcome Ali
```

Another student says:

```
My name is Ahmed
```

The same office responds:

```
Welcome Ahmed
```

The office remains the same, but the information changes.

Dynamic routes work in the same way.

The route remains the same, but the value changes.

# Normal Route vs Dynamic Route

## Normal Route

Example:

```python
@app.route("/student")
def student():
    return "Student Page"
```

URL:

```
/student
```

Output:

```
Student Page
```

This route always returns the same response.

---

## Dynamic Route

Example:

```python
@app.route("/student/<name>")
def student(name):
    return "Hello " + name
```

URL:

```
/student/Ali
```

Output:

```
Hello Ali
```

URL:

```
/student/Ahmed
```

Output:

```
Hello Ahmed
```

The same function handles different values.

---

# Understanding Dynamic Route Structure

Example:

```
http://127.0.0.1:5000/student/Noor
```

Breaking it down:

```
127.0.0.1  → Computer Address

5000       → Flask Server Port

/student   → Route

Noor       → Dynamic Value
```

Here:

```
/student
```

is fixed.

But:

```
Noor
```

can change.

---

# Complete Dynamic Route Example

```python
from flask import Flask

app = Flask(__name__)


@app.route("/student/<name>")
def student(name):
    return "Welcome " + name


app.run(debug=True)
```

---

# First Request Example

Browser:

```
http://127.0.0.1:5000/student/Ali
```

Flask checks:

```
/student/<name>
```

The route matches.

Flask automatically assigns:

```
name = Ali
```

Then Flask executes:

```python
student("Ali")
```

The response is:

```
Welcome Ali
```

---

# Second Request Example

Browser:

```
http://127.0.0.1:5000/student/Noor
```

Flask assigns:

```
name = Noor
```

Function execution:

```python
student("Noor")
```

Response:

```
Welcome Noor
```

The same route handles different users.

---

# How Dynamic Routing Works Internally

The complete flow:

```
Browser

     |
     |
Request:
/student/Ali

     |
     |
Flask Router

     |
     |
Find matching route:

/student/<name>

     |
     |
Extract value:

name = Ali

     |
     |
Execute:

student("Ali")

     |
     |
Return Response

     |
     |
Browser Display
```

---

# Multiple Dynamic Values

A route can contain more than one dynamic value.

Example:

```python
@app.route("/student/<name>/<age>")
def student(name, age):
    return name + " is " + age + " years old"
```

URL:

```
http://127.0.0.1:5000/student/Noor/24
```

Flask extracts:

```
name = Noor

age = 24
```

Function execution:

```python
student("Noor", "24")
```

Output:

```
Noor is 24 years old
```

# Real-World Examples of Dynamic Routes

## 1. YouTube

YouTube has millions of videos.

It does not create a separate route for every video.

Instead, it uses a dynamic route concept.

Example:

```
/watch/<video_id>
```

The video ID changes for every video.

Example:

```
youtube.com/watch?v=abc123
```

The route remains the same, but the video ID changes.

---

## 2. Instagram Profiles

Instagram users have different usernames:

```
instagram.com/noor

instagram.com/cristiano
```

Instagram does not create separate code for every user.

It uses a dynamic route:

```
/<username>
```

The username changes dynamically.

---

## 3. E-Commerce Websites

Shopping websites have thousands of products.

Example:

```
amazon.com/product/iPhone15

amazon.com/product/Laptop
```

A dynamic route can handle this:

```
/product/<product_name>
```

The product name changes.

# Dynamic Route Data Types

Flask allows different types of dynamic values.

---

## 1. String (Default)

Example:

```python
@app.route("/user/<name>")
def user(name):
    return name
```

URL:

```
/user/Ali
```

Value:

```
name = Ali
```

---

## 2. Integer

Example:

```python
@app.route("/user/<int:id>")
def user(id):
    return str(id)
```

URL:

```
/user/101
```

Value:

```
id = 101
```

---

## 3. Float

Example:

```python
@app.route("/price/<float:value>")
def price(value):
    return str(value)
```

URL:

```
/price/99.5
```

Value:

```
value = 99.5
```

---

# Important Rule

The dynamic variable name and the function parameter name must be the same.

Correct:

```python
@app.route("/student/<name>")
def student(name):
    return name
```

Here:

```
<name>
```

matches:

```
name
```

---

Wrong:

```python
@app.route("/student/<name>")
def student(username):
    return username
```

This will create a problem because Flask sends the value using the name `name`.

---

# Dynamic Routes Summary

| Concept            | Meaning                              |
| ------------------ | ------------------------------------ |
| Dynamic Route      | A route that accepts changing values |
| `<name>`           | Dynamic variable inside URL          |
| One Route          | Handles many different values        |
| Function Parameter | Receives dynamic data                |
| Benefit            | Reduces duplicate code               |

---

# Final Concept

Dynamic Routing means:

> Creating one URL pattern that can handle different values.

Example:

```
/student/Ali
        ↓
Hello Ali


/student/Ahmed
        ↓
Hello Ahmed


/student/Noor
        ↓
Hello Noor
```

All requests use the same route:

```python
/student/<name>
```

Dynamic Routes make Flask applications cleaner, shorter, and easier to maintain.
