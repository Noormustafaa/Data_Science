## Python POST Request Example

```python
import requests

# 1. The URL where we want to send the data
url = "https://jsonplaceholder.typicode.com/posts"

# 2. The data we want to send (Python Dictionary)
apna_data = {
    "title": "My First Post",
    "body": "Learning APIs is very easy!",
    "userId": 1
}

# 3. Sending the POST request
response = requests.post(url, json=apna_data)

# 4. Checking the server's response
print("Status Code:", response.status_code)
print("Server Response:")
print(response.json())

```

---

## Understanding the Code

Let's break down the logic:

`url`**: The destination address of the server where we are sending the data.
`apna_data`**: In a POST request, we always send a **Body** (the information we want to save). Here, we defined a title, body, and user ID.
* **`requests.post(..., json=apna_data)`**:
* In a GET request, we only use `requests.get()` because we aren't sending anything.
* In a POST request, we use `.post()` and include `json=apna_data` to ensure Python converts our dictionary into JSON format, which is the standard language for the internet.


* **`response.status_code`**: When the server receives the data, it sends back a code. A **201** status code means **"Created"**—it confirms that your data reached the server and was successfully saved.
* **`response.json()`**: The server doesn't just send a code; it returns a confirmation message (usually the data you sent, plus a new `id` field) so you know the task was completed.

**Real-World Example:** When you write a status on Facebook and click "Post," a POST request is running in the background, sending your text to Facebook's server.
